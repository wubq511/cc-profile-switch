# Task: Backup and Edit Commands

## Description

Implement `ccps backup <name>` and `ccps edit <name> [file]` with strict path boundaries and clear file mappings.

## Acceptance Criteria

- [ ] `ccps backup <name>` copies the selected profile into `backups\<name>-YYYYMMDD-HHmmss`.
- [ ] Backup includes `profile.json`, `claude-home`, `mcp.json`, and `plugins`.
- [ ] Backup does not modify the source profile.
- [ ] `ccps edit <name>` opens the selected profile directory.
- [ ] `ccps edit <name> CLAUDE.md` opens `claude-home\CLAUDE.md`.
- [ ] `ccps edit <name> settings.json` opens `claude-home\settings.json`.
- [ ] `ccps edit <name> mcp.json` opens profile-level `mcp.json`.
- [ ] `ccps edit <name> profile.json` opens profile-level `profile.json`.
- [ ] Any other edit target is rejected without opening a path.
- [ ] Tests cover backup timestamp format, backup contents, approved edit mappings, and rejected path traversal.

## Technical Details

- Implement editor/default opener behavior in `src/platform/editor.ts`.
- Use stable timestamp formatting for backup directory names.
- Do not prompt to create arbitrary files outside the approved mapping.
- Never open or edit `C:\Users\h\.claude`.
- Keep process launch for editor separate from Claude Code launch logic.

## Dependencies

- Depends on `4` for profile path resolution and template structure.
