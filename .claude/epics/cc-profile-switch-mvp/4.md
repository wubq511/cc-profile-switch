# Task: App Config and Profile Templates

## Description

Implement the app-home configuration layer and reusable profile template generation using the final `claude-home` profile structure.

## Acceptance Criteria

- [x] App home defaults to `%USERPROFILE%\.cc-profile-switch`.
- [x] `config.json` can be created, loaded, parsed, and saved safely.
- [x] Template data exists for `coding`, `study`, `work`, `research`, `general`, and `blank`.
- [x] Template output includes `profile.json`, `claude-home\CLAUDE.md`, `claude-home\settings.json`, `claude-home\skills`, `claude-home\agents`, `mcp.json`, and `plugins`.
- [x] File creation is non-overwriting by default.
- [x] Tests use temporary directories and never touch the real app home or real Claude Code config.

## Technical Details

- Implement `src/core/app-config.ts` for app-home and config operations.
- Implement `src/core/profile-template.ts` for profile template materialization.
- Template JSON files must be valid JSON objects.
- Template text should make clear that profile config is user-level global config for Claude Code.
- Keep timestamps deterministic in tests by injecting time where useful.
