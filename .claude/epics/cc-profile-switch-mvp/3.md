# Task: Windows Path, Errors, and Schemas

## Description

Implement the foundational utilities that every later profile command depends on: Windows path resolution, containment checks, safe profile name validation, structured errors, and Zod schemas for app and profile configuration.

## Acceptance Criteria

- [x] `src/platform/windows-path.ts` resolves user home, app home, and child paths with Node `path` APIs.
- [x] Path containment checks block traversal outside the expected base directory.
- [x] Profile names allow only safe characters and reject traversal, empty names, path separators, and reserved ambiguous values.
- [x] `src/utils/errors.ts` or equivalent provides structured errors with user-facing next-step guidance.
- [x] Zod schemas exist for `config.json` and `profile.json`.
- [x] Unit tests cover valid and invalid profile names, Windows path containment, traversal attempts, and schema parsing failures.

## Technical Details

- All path helpers should return absolute normalized paths.
- Containment checks should be case-insensitive where appropriate for Windows paths.
- Schemas should validate the launch config shape, including `mcpMode`, `pluginDirs`, and `disableAutoMemory`.
- Avoid relying on shell string parsing for any path logic.

- Depends on `2` because the project structure and test runner must exist first.
