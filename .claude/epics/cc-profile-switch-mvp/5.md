# Task: Init and Create Commands

## Description

Implement `ccps init` and `ccps create <name> --template <template>` so users can create the app home, default profiles, and additional profiles from templates.

## Acceptance Criteria

- [x] `ccps init` creates app home, `config.json`, `profiles`, `backups`, and the five default profiles.
- [x] Re-running `ccps init` preserves user-edited files and does not overwrite existing profiles.
- [x] `ccps create <name> --template <template>` supports `coding`, `study`, `work`, `research`, `general`, and `blank`.
- [x] `ccps create` rejects invalid profile names.
- [x] `ccps create` refuses to overwrite an existing profile.
- [x] Command output includes concise next-step guidance.
- [x] Integration tests cover fresh init, repeated init, successful create, duplicate create, and invalid names.

## Technical Details

- Add command handlers under `src/commands/`.
- Add profile creation logic under `src/core/profile.ts` or a focused profile service.
- Respect the exact `claude-home` structure from the PRD.
- Do not read from or copy `C:\Users\h\.claude`.
- Keep output readable and low-noise.
