# Task: Real Launch and README

## Description

Implement real `ccps launch <name>` process spawning after validation, dry-run, and manual verification are in place. Finish the MVP README so a Windows user can run the full workflow safely.

## Acceptance Criteria

- [ ] Real launch validates the selected profile before spawning Claude Code.
- [ ] Real launch uses `spawn('claude', args, { cwd, stdio: 'inherit', shell: false, env })`.
- [ ] Real launch sets `CLAUDE_CONFIG_DIR` to the selected profile's `claude-home`.
- [ ] Real launch keeps cwd as the current project directory unless a safe explicit cwd option is provided.
- [ ] Real launch never uses `--add-dir` for the current project.
- [ ] Real launch never changes cwd to `.cc-profile-switch` or any runtime directory.
- [ ] Launch failure messages are clear and include next-step guidance.
- [ ] Last-used metadata is updated only after a valid launch attempt reaches the intended execution point.
- [ ] README documents install, local development, commands, profile structure, safety boundaries, dry-run, launch behavior, and verification steps.
- [ ] `npm run check` passes.

## Technical Details

- Keep real process spawning in `src/platform/process.ts` and core launch decisions in `src/core/launcher.ts`.
- Do not build shell command strings for execution.
- Preserve the default MCP merge behavior.
- If verification showed Claude Code limitations, document them and reflect them in launch warnings.
- README must clearly state that the tool does not migrate OAuth/session/token/history/cache.
