# Task: Project Setup and CLI Skeleton

## Description

Create the TypeScript Node.js CLI foundation for `ccps` without implementing profile business logic or launch behavior. The goal is a buildable, testable project skeleton with Commander wired to a safe `--help` experience.

## Acceptance Criteria

- [x] `package.json` defines the `ccps` binary as `dist/index.js`.
- [x] Scripts exist for `dev`, `build`, `start`, `test`, `test:watch`, `lint`, `format`, and `check`.
- [x] TypeScript, tsup, Vitest, ESLint, and Prettier are configured.
- [x] `src/index.ts` initializes Commander and shows useful help text.
- [x] Source directories exist: `commands`, `core`, `platform`, `schemas`, `templates`, and `utils`.
- [x] Running `npm run build` produces `dist/index.js`.
- [x] Running the CLI with `--help` has no profile filesystem side effects.

## Technical Details

- Use TypeScript and CommonJS output through tsup.
- Keep command registration minimal; placeholder command modules may exist, but should not perform real profile operations yet.
- Do not read from, copy, write, or inspect `C:\Users\h\.claude`.
- Do not implement `launch` process spawning in this task.
- Prefer small module boundaries that match the epic architecture.
