# Task: Launch Plan and Dry Run

## Description

Implement launch-plan construction and `ccps launch <name> --dry-run` without spawning Claude Code. This task proves the launch behavior before real process execution exists.

## Acceptance Criteria

- [ ] `ccps launch <name> --dry-run` validates the profile before building the plan.
- [ ] Dry-run output includes profile name, profile path, `claude-home`, cwd, MCP mode, plugin dirs, command, args, env changes, warnings, and validation result.
- [ ] The launch plan sets `CLAUDE_CONFIG_DIR` to the selected profile's `claude-home`.
- [ ] The launch plan keeps cwd as the current working directory by default.
- [ ] The launch plan supports an explicit safe `--cwd` option if implemented.
- [ ] Default MCP mode passes `--mcp-config <profile>\mcp.json` when present and does not pass `--strict-mcp-config`.
- [ ] Strict MCP is only added for explicit strict mode.
- [ ] Dry-run never spawns Claude Code.
- [ ] Tests cover env construction, cwd handling, MCP args, strict mode, off mode, plugin dirs, validation failure blocking, and output content.

## Technical Details

- Implement `src/core/launcher.ts` with `buildLaunchPlan()` and dry-run formatting.
- Add `src/platform/process.ts` only for command lookup if needed; do not execute `spawn` yet.
- Use args arrays internally. Only stringify command lines for display.
- Include a clear project config message: project config is preserved because Claude starts in the current cwd.
- Do not implement real launch in this task.
