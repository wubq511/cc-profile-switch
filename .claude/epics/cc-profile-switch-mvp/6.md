# Task: List, Show, and Validate Commands

## Description

Implement profile inspection and validation: `ccps list`, `ccps show <name>`, and `ccps validate <name>`. These commands establish the safety gate that later blocks unsafe launches.

## Acceptance Criteria

- [x] `ccps list` shows profile names, status, last-used value when available, and description.
- [x] `ccps list` clearly prompts for `ccps init` when the app home or profiles directory is missing.
- [x] `ccps show <name>` displays absolute profile path, `claude-home` path, required file status, JSON validation status, and project config preservation notes.
- [x] `ccps validate <name>` reports `valid`, `warning`, or `error`.
- [x] Validation detects missing required files/directories, invalid JSON, invalid manifest shape, path traversal, and sensitive filenames.
- [x] High-risk sensitive names block launch-level validation; medium-risk names produce warnings.
- [x] Tests cover valid profiles, missing files, invalid JSON, sensitive filename scan, and uninitialized app home behavior.

## Technical Details

- Implement `src/core/validator.ts` as reusable business logic, not just command output code.
- Validation findings should include severity, code, message, path when relevant, and suggested fix.
- Required files: `profile.json`, `claude-home\CLAUDE.md`, `claude-home\settings.json`, `mcp.json`.
- Required directories: `claude-home`, `claude-home\skills`, `claude-home\agents`, and `plugins`.
- Sensitive scan high-risk keywords: `.claude.json`, `token`, `secret`, `credential`, `credentials`, `session`, `oauth`.
- Sensitive scan medium-risk keywords: `history`, `cache`, `log`, `transcript`.
