# Task: TUI Terminal Adapter

## Description

Wire `ccps tui` to the TUI controller through a Windows-friendly terminal adapter. The adapter should provide a usable keyboard-first interaction while keeping business behavior inside the shared controller and core services.

## Acceptance Criteria

- [ ] `ccps tui` starts the TUI flow from Commander.
- [ ] The terminal adapter implements the controller prompt/output ports.
- [ ] The selected prompt/readline approach works in Windows Terminal and PowerShell.
- [ ] Any added TUI dependency is small, maintained, documented in the implementation, and wrapped behind the adapter boundary.
- [ ] The TUI can exit cleanly without mutating state.
- [ ] TUI wiring tests cover command registration and adapter invocation without requiring a real interactive terminal.

## Technical Details

- Prefer Node built-in `readline` primitives unless implementation proves they are insufficient for the required selection UX.
- Keep adapter code isolated from core services.
- Do not start a real long-running Claude process from the TUI in V0.2; use launch dry-run output.
