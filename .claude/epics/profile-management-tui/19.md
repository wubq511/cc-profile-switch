# Task: Default Launch Resolution

## Description

Allow `ccps launch` to omit the profile argument when `config.json.defaultProfile` is set, while preserving all existing launch and dry-run behavior.

## Acceptance Criteria

- [x] `ccps launch [profile]` accepts an optional profile argument.
- [x] `ccps launch --dry-run` resolves to `defaultProfile` when no profile argument is provided.
- [x] `ccps launch` resolves to `defaultProfile` when no profile argument is provided.
- [x] Launch with an explicit profile still takes precedence over default.
- [x] Launch without an explicit profile and without a default fails clearly with next-step guidance.
- [x] Launch fails clearly if `defaultProfile` points to a missing profile.
- [x] Existing dry-run parity, cwd handling, MCP merge behavior, API env behavior, plugin directory behavior, skip-permissions default, and memory isolation remain unchanged.

## Technical Details

- Prefer a shared `resolveLaunchProfile` core helper used by both `buildLaunchPlan` and `launchProfile`.
- Keep `formatLaunchDryRun` output explicit about which profile was selected.
- Update existing launcher tests rather than replacing them.
