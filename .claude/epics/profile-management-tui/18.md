# Task: CLI Profile Management Commands

## Description

Add the V0.2 CLI commands for profile management while keeping Commander handlers thin. Commands should parse arguments, collect any needed confirmation, call core services, and print compact actionable output.

## Acceptance Criteria

- [x] `ccps copy <from> <to>` calls the shared copy service and reports source, target, and next suggested command.
- [x] `ccps rename <old> <new>` calls the shared rename service and reports updated config references when relevant.
- [x] `ccps remove <name>` prompts for exact-name confirmation, calls the shared remove service, reports backup path and deletion result, and has no `--force` or `--yes` bypass.
- [x] `ccps default [name]` shows the current default when no name is provided.
- [x] `ccps default <name>` sets an existing profile as default.
- [x] `ccps default --clear` clears the default profile.
- [x] CLI help includes the new commands.
- [x] CLI lifecycle tests cover success and failure behavior using injected prompt/confirmation runtime methods.

## Technical Details

- Extend `CommandRuntime` with an injectable confirmation/input method rather than reading directly from stdin inside command handlers.
- Keep existing output style: short status lines plus next steps.
- Use the same error formatting flow as the current CLI.
- Do not duplicate core filesystem or config logic inside `src/commands/index.ts`.
